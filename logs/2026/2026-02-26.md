# 2026-02-26 TIL - 보안 관점 CS 핵심 메모

## 오늘의 핵심
- 비밀값은 코드에 두지 않고 `env.example`에는 "키 이름/설명"만 남긴다.
- 기본 정책은 `허용`이 아니라 `거부`로 시작한다(Fail-safe default).
- 권한은 역할 단위로 쪼개고, 필요한 범위만 부여한다(Least Privilege).
- 입력값 검증은 `타입 -> 길이 -> 허용값` 순서로 서버에서 강제한다.
- 로그는 추적에 충분해야 하지만, 토큰/비밀번호는 반드시 마스킹한다.

## 왜 중요한가
보안 사고 대부분은 "고급 해킹 기법"보다 기본 원칙 누락에서 시작한다.  
실무에서는 새 기술보다, 같은 실수를 반복하지 않게 만드는 운영 기준이 먼저다.

## 내일 적용할 것
- 신규 API 1개를 골라 인증/인가, 입력 검증, 로그 마스킹 항목으로 셀프 점검하기.
