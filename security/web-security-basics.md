# 웹 보안 기본 점검표 (실무형 요약)

## 1. 인증/인가
- 인증 실패와 인가 실패를 분리한다 (`401` vs `403`)
- 민감 API는 "로그인 여부 + 리소스 소유권" 둘 다 확인한다
- JWT는 짧은 Access + 재발급 가능한 Refresh로 운영한다

## 2. 입력 검증
- 타입 검증: 숫자/문자/날짜 형식 강제
- 길이 제한: 과도한 payload 차단
- 화이트리스트: enum 필드는 허용값만 통과
- 서버 기준 검증: 프론트 검증은 UX용, 보안 기준은 서버에서 확정

## 3. 대표 취약점 체크
### XSS
- 사용자 입력을 그대로 렌더링하지 않는다
- 템플릿 엔진 escape 기본값 유지
- HTML 허용 시 sanitizer 필수

### CSRF
- 쿠키 기반 인증이면 CSRF 토큰 사용
- SameSite 설정 확인
- 상태 변경 API(`POST/PUT/PATCH/DELETE`) 보호 적용

### SQL Injection
- ORM/파라미터 바인딩 사용
- 문자열 연결 쿼리 금지
- 검색/정렬 파라미터 화이트리스트 처리

### SSRF
- URL 입력을 그대로 서버 요청으로 사용하지 않음
- 내부망 IP 대역 접근 차단
- 외부 요청 타임아웃/허용 도메인 정책 적용

## 4. 운영 점검 루틴
- 에러 로그에서 인증 실패 패턴 확인
- 비정상 트래픽(짧은 시간 반복 요청) 모니터링
- 릴리즈 전 보안 체크리스트와 API 명세 동기화

## 5. 면접용 한 줄 정리
보안은 취약점을 "찾는 일"보다, 같은 실수가 반복되지 않게 구조와 규칙을 만드는 일이다.
